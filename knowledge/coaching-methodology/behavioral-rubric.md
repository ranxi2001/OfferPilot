# 行为面试评分 Rubric：STAR 五维量化评估

> 来源：swe-interview-coach 评分体系 + 多个面试辅导平台的评分共识。行为面试占面试总评的 20-40%，但多数候选人完全没有准备。

---

## STAR 回答五维评分（每维 1-5 分）

### 1. STAR 清晰度 (Clarity)

| 分数 | 标准 |
|------|------|
| 5 | S/T/A/R 四段清晰，每段有具体细节，逻辑链完整 |
| 4 | 四段基本清晰，个别段落可以更具体 |
| 3 | 结构可辨认，但某段含糊或跳跃 |
| 2 | 结构混乱，S/T 混在一起或 R 缺失 |
| 1 | 无结构，像流水账或只有结论 |

**升级方法**: 开头主动说"我举一个 X 场景的例子，当时情况是...我的任务是...我做了...结果是..."

### 2. 量化指标 (Metrics)

| 分数 | 标准 |
|------|------|
| 5 | 多个具体数字，前后对比清晰（"从 X 到 Y，提升 Z%"） |
| 4 | 有 1-2 个关键数字 |
| 3 | 有模糊量化（"大幅提升"、"显著改善"） |
| 2 | 偶尔提到规模但无对比 |
| 1 | 完全没有数字或量化 |

**升级方法**: 任何成果都可以量化 — 时间节省、覆盖用户数、错误率降低、团队规模、项目周期

### 3. 简洁性 (Conciseness)

| 分数 | 标准 |
|------|------|
| 5 | 2-3 分钟内讲完，信息密度高，没有废话 |
| 4 | 3-4 分钟，偶有冗余但整体紧凑 |
| 3 | 4-5 分钟，有些绕路但最终回到主题 |
| 2 | 超过 5 分钟，明显跑题或重复 |
| 1 | 讲不完或完全跑题 |

**升级方法**: 预先准备好每个故事的 2 分钟版和 4 分钟版，面试时先给短版，面试官追问再展开

### 4. 价值观对齐 (Values Alignment)

| 分数 | 标准 |
|------|------|
| 5 | 故事自然体现目标公司核心价值观（如 ownership、customer obsession） |
| 4 | 体现了通用正向品质（主动性、责任心） |
| 3 | 中性，没有负面但也没亮点 |
| 2 | 暴露了一些红旗（推卸责任、不尊重同事） |
| 1 | 明确负面信号 |

**升级方法**: 面试前研究目标公司的 leadership principles / 价值观，选择能体现这些品质的故事

### 5. 反思深度 (Reflection)

| 分数 | 标准 |
|------|------|
| 5 | 主动说"如果重来我会..."或"这件事教会我..."，有真实反思 |
| 4 | 在追问下能给出反思 |
| 3 | 承认有改进空间但无具体方向 |
| 2 | 完全正面叙述，没有反思 |
| 1 | 对明显失误毫无觉察 |

**升级方法**: 每个故事结尾主动加一句"回顾来看，我会在 X 环节做得更好"

## 行为面常见题型与故事映射

### 必备 5 个核心故事

| 故事 | 覆盖题型 | STAR 要点 |
|------|---------|----------|
| 最大技术挑战 | "最难的项目"、"克服困难"、"技术决策" | S: 约束条件 T: 必须解决的问题 A: 技术方案选择过程 R: 量化成果 |
| 团队冲突/协作 | "意见不合"、"推动他人"、"cross-team" | S: 谁和谁冲突 T: 项目进度受阻 A: 沟通策略 R: 达成共识 |
| 失败与反思 | "犯过的错"、"最大弱点"、"后悔的决定" | S: 当时判断 T: 做错了什么 A: 如何补救 R: 学到了什么 |
| 超出预期 | "exceed expectations"、"go above and beyond" | S: 常规要求 T: 主动发现机会 A: 额外付出 R: 超预期结果 |
| 时间压力 | "deadline"、"prioritization"、"多任务" | S: 多个紧急任务 T: 必须取舍 A: 优先级判断 R: 按时交付 |

### Amazon Leadership Principles 映射

| LP | 适合的故事类型 |
|----|-------------|
| Customer Obsession | 用户反馈驱动的功能改进 |
| Ownership | 非职责范围内主动承担的事 |
| Dive Deep | 深入排查 root cause 的经历 |
| Bias for Action | 信息不完整时快速决策 |
| Earn Trust | 承认错误、透明沟通的场景 |
| Deliver Results | 克服障碍按时交付 |

## 行为面红旗检测

回答中出现以下信号会被扣分：

| 红旗 | 面试官解读 | 修正方式 |
|------|-----------|---------|
| "这不是我的问题" | 缺少 ownership | 说"我可以做得更好的是..." |
| "我们团队做了..." | 个人贡献不清楚 | 用"我"替代"我们"，明确自己的角色 |
| 没有 Result | 只讲过程不讲结果 | 用数字收尾 |
| 讲了 10 分钟 | 缺乏结构化表达能力 | 主动控时，给面试官追问的空间 |
| 编造感 | 细节经不起追问 | 只讲真实经历，不确定的说"具体数字我需要确认" |

## 系统设计面评分六轴

| 轴 | 1-5 评分标准 |
|---|---|
| 需求澄清 | 是否主动问约束（QPS、一致性、可用性） |
| 高层架构 | 核心组件是否完整、数据流是否清晰 |
| 深入设计 | 对核心模块的实现细节是否了解 |
| 权衡取舍 | 是否讨论了方案的优劣和替代方案 |
| 可扩展性 | 是否考虑了规模增长后的架构演进 |
| 沟通能力 | 表达是否清晰、能否引导面试官 |
