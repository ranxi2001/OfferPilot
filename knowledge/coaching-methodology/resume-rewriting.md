# 简历包装方法论：方向感知的经历重写

> 核心原则：不同岗位看同一段经历的视角完全不同。同一个项目，Agent 开发岗看系统架构，后端岗看 API 设计，量化岗看算法迭代。

---

## 方向感知包装策略

### Agent 开发方向

**核心叙事**: 从0到1构建 Agent 系统，擅长设计 Agent 架构、开发 Tool/Function、编排多步骤工作流、实现端到端自动化。

**强调维度**:

| 维度 | 包装方式 | 示例句式 |
|------|---------|---------|
| 系统架构 | Agent 如何分层、状态如何流转 | "设计5阶段 Agent Pipeline，状态机驱动的多轮排查流程" |
| Tool/Function 开发 | 开发了哪些 Tool | "开发12个 Tool Function 对接 Hive/Presto/风控 API" |
| 编排与调度 | 条件路由、并行执行 | "Agent 根据中间结果动态编排 SQL 执行顺序，支持5路并行" |
| 自动化率 | 人工→自动对比 | "将40分钟人工排查压缩至4分钟全自动执行(10x)" |
| Prompt Engineering | 链式推理、结构化输出 | "设计 Prompt Chain 实现三级推理" |
| 从0到1 | 独立设计整个系统 | "从零构建市场监察 Agent 系统，覆盖全链路" |

**黄金句式**:
- "从零设计并实现了..."
- "Agent 自动编排..."
- "开发 N 个 Tool Function 覆盖..."
- "将人工流程 X 分钟压缩至 Y 分钟(Nx 效率提升)"
- "端到端无人值守完成..."

### 后端开发方向

**核心叙事**: 能设计高性能数据系统的后端工程师，擅长查询优化、API 设计、工具链开发。

**强调维度**:
- API 设计与工程架构（RESTful、中间件、错误处理）
- 数据库与查询优化（SQL 优化、OOM 规避、分片策略）
- 性能指标（QPS、延迟、数据量级）
- 系统可靠性（重试、断点续传、连接池）

### 数据分析方向

**核心叙事**: 从海量数据中发现洞察并驱动决策。

**强调维度**:
- 分析方法论（假设→验证→结论）
- 数据规模与复杂度
- 业务影响（驱动了什么决策、节省了多少）
- 可视化与报告产出

### AI 算法方向

**核心叙事**: 模型训练与优化专家。

**强调维度**:
- 模型选型与微调经验
- 量化成果指标（F1-Score、准确率）
- 数据工程（清洗、标注、增强）
- 部署与推理优化

## Title 包装规则

原始 title 对目标岗位没吸引力，要改成匹配方向的 title：

| 原始Title | Agent方向 | 后端方向 | 数据方向 |
|-----------|-----------|---------|---------|
| 软件开发实习生 | AI Agent 开发工程师(实习) | 后端开发工程师(实习) | 数据工程师(实习) |
| 产品研发工程师 | 全栈 Agent 研发 | 后端开发工程师 | 数据平台工程师 |
| 市场监察实习生 | 风控 Agent 算法工程师(实习) | 大数据开发工程师(实习) | 数据分析工程师(实习) |

**原则**: Title 要让 HR 一眼看出与目标岗位的匹配度。

## 量化包装技巧

### 必做
- 每段经历至少 1 个量化指标
- 优先用百分比和倍数："提升 40%"、"10x 效率提升"
- 给出规模感："覆盖 100+ 工具"、"日处理 50 万条"

### 转化方式
| 模糊表述 | 量化重写 |
|---------|---------|
| "大幅提升性能" | "将延迟从 2s 优化至 200ms (10x)" |
| "处理海量数据" | "日处理 500 万条交易记录" |
| "效果显著提升" | "F1-Score 从 72% 提升至 96%" |
| "获得好评" | "上线首周 DAU 增长 35%" |

### 弱化/不写
- 与目标方向无关的细节
- 纯行政性工作描述
- 无法在面试中展开的虚数据

## Bullet Point 结构

### Impact-First 格式（推荐）
```
[成果] 通过 [方法] 实现 [具体做法]
```

### 标准格式
```
动作 + 对象 + 结果（量化）
```

### 反面示例
```
❌ 负责了 Agent 系统的开发和维护工作
✅ 从零设计并实现了多阶段 Agent Pipeline，将人工排查流程从 40 分钟压缩至 4 分钟
```

---

## 内容精简原则

- 每段经历 3-4 个 bullet points
- 目标 1 页（高年资可 2 页）
- 最相关的经历放最前
- 不相关的技能/经历大胆删除
- 每个 bullet point 必须有一个加粗亮点
