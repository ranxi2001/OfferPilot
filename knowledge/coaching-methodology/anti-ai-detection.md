# 反 AI 检测：让简历和回答听起来像人写的

> 来源：ai-resume-humanizer-prompts 项目研究。越来越多 HR 和 ATS 系统会标记"AI 生成感"过强的简历。

---

## AI 生成内容的 5 种失败模式

### 1. Buzzword 堆砌 (Buzzword Stacking)

**症状**: 每句话都塞满流行词，但缺少具体语境

```
❌ "利用前沿 AI 技术赋能业务，驱动数字化转型，打造智能化解决方案"
✅ "用 GPT-4 + RAG 搭了内部知识问答系统，客服工单处理时间从 15 分钟降到 3 分钟"
```

### 2. 对称句式 (Symmetric Bullet Rhythm)

**症状**: 每条 bullet point 长度、结构几乎相同

```
❌ "设计并实现了 X 系统，提升了 Y 效率"
   "设计并实现了 A 平台，优化了 B 流程"
   "设计并实现了 C 工具，改善了 D 体验"

✅ 变化句式长度和结构：
   "主导 X 系统架构设计，3 周内从零上线"
   "发现 Y 性能瓶颈后独立重构 SQL 层——延迟从 2s 降到 200ms"
   "推动团队采用 Z 方案，替代了维护 2 年的旧系统"
```

### 3. 模糊归属 (Vague Ownership)

**症状**: 无法判断是本人做的还是团队做的

```
❌ "参与了系统的设计与开发工作"
✅ "我负责其中的消息队列模块，从选型(对比 Kafka/RabbitMQ)到上线"
```

### 4. 万能摘要 (Universal Summary)

**症状**: 把名字和年限改掉后，适用于任何人

```
❌ "资深工程师，拥有丰富的项目经验和优秀的沟通能力"
✅ "3 年 Agent 系统开发经验，主导过 2 个从零到一的 LLM 产品上线"
```

### 5. 虚构指标 (Fabricated Metrics)

**症状**: 数字太整、太好看，经不起追问

```
❌ "将系统性能提升 500%"（面试追问 how → 答不上来）
✅ "通过引入 Redis 缓存热点查询，P99 延迟从 1.2s 降到 180ms"（可以详细解释）
```

## Buzzword 黑名单

以下词汇在简历中出现过多会触发"AI 生成"判断：

| 避免使用 | 替代方案 |
|---------|---------|
| 赋能 | 支撑/服务/帮助 |
| 闭环 | 从 X 到 Y 完整链路 |
| 抓手 | 具体方法/工具名 |
| 沉淀 | 总结为文档/建立了规范 |
| 对齐 | 和 X 团队确认 |
| 拉通 | 组织了 X 人参加评审 |
| 颗粒度 | 说清楚具体到什么级别 |
| 方法论 | 说具体是什么方法 |
| 打法 | 策略/方案 |
| 底层逻辑 | 核心原因 |

## 自然化改写规则

### 规则 1：绝不虚构数字
如果不记得确切数字，用范围或相对量：
- "约 200 个" 比 "200 个" 更可信
- "提升了 2-3 倍" 比 "提升 300%" 更自然

### 规则 2：字数增长不超过 10%
改写后的长度不应显著超过原文，否则 = 注水

### 规则 3：具体名词 > 抽象动词
- ❌ "优化了数据处理流程"
- ✅ "用 Spark 替换了原来的单机 pandas 脚本"

### 规则 4：保留口语感
简历不是论文，适度的非正式表达更真实：
- "说白了就是..." / "踩了个坑..." / "后来发现..."（在面试中使用）
- 简历中用"独立"、"从零"这类有画面感的词

### 规则 5：每条 bullet 有独特细节
至少包含一个只有做过这件事的人才知道的细节：
- 具体的工具版本/配置
- 遇到的具体错误/异常
- 选型时被否掉的备选方案

## 面试回答的自然化

### AI 腔 vs 人话

| AI 腔 | 人话 |
|-------|------|
| "首先...其次...最后..." | "最关键的一点是...另外还有个有意思的..." |
| "这是一个很好的问题" | 直接回答 |
| "从以下几个维度来分析" | "我觉得主要看两个东西" |
| "综上所述" | "所以总结下来就是..." |
| "让我从 X 角度来回答" | 直接说内容 |

### 自然回答的标志

- 有犹豫和自我修正："嗯...准确说应该是..."
- 有第一人称经验："我当时遇到这个问题时..."
- 有不确定承认："这块我没太深入研究，但我理解的是..."
- 有反问确认："你说的是 X 场景还是 Y 场景？"

---

## 简历自检 Checklist

- [ ] 没有连续 3 条以上句式相同的 bullet
- [ ] 每条 bullet 有至少 1 个具体名词（工具名/框架名/数据量）
- [ ] 数字都是面试中能解释清楚的
- [ ] 没有出现黑名单词汇超过 2 次
- [ ] 读出来像是"这个人在和我聊天"而非"这是 ChatGPT 写的"
