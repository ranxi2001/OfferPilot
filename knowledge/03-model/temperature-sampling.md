# 模型能力 - Temperature 与采样策略

## Q：Temperature、Top-P、Top-K 分别控制什么？Agent 场景该怎么设？

> 来源：某 AI 应用团队面试

**新手答**："Temperature 越高越随机，越低越确定"

**高手答**：

这三个参数都在控制 token 采样的随机性，但作用层面不同：

1. **Temperature**：缩放 logits 后再 softmax。T=0 时 argmax（贪婪），T>1 时分布变平（更随机）。本质上是调整概率分布的"锐度"。

2. **Top-P (Nucleus Sampling)**：按概率降序累加，取累计概率到 P 的 token 子集，再在子集内按比例采样。自适应地控制候选集大小——高确信时候选少，低确信时候选多。

3. **Top-K**：固定取概率最高的 K 个 token。简单但不够灵活——有时 top-2 就够了，有时需要 top-100。

Agent 场景的设置策略：

- **Tool calling / 结构化输出**：T=0 或极低。工具名和 JSON 参数不能有创意。
- **对话生成**：T=0.7-0.9。需要自然但不跑偏。
- **创意写作 / brainstorm**：T=1.0+。需要多样性。
- **代码生成**：T=0.2-0.4。需要正确性但允许一定灵活度。

工程注意：
- Anthropic 的 API 中 temperature 默认 1.0，OpenAI 默认也是 1.0
- 不要同时设 top_p 和 temperature 到极端值，效果不可预测
- 对于需要可复现的场景（测试、评估），用 T=0 + seed 固定随机种子
