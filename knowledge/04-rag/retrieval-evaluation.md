# RAG - 检索质量评估

## Q：如何评估 RAG 系统的检索质量？有哪些指标和方法？

> 来源：某搜索团队面试

**新手答**："看用户反馈好不好"

**高手答**：

RAG 评估要分两层：检索层和生成层，不能混为一谈。

**检索层指标**：

1. **Recall@K**：相关文档在 top-K 结果中被召回的比例。K 通常取 5-20。
2. **MRR (Mean Reciprocal Rank)**：第一个相关文档的排名倒数的平均值。关注"第一个命中多快"。
3. **NDCG**：归一化折损累积增益，考虑相关性的分级（高度相关 vs 部分相关）。
4. **Hit Rate**：至少命中一个相关文档的查询比例。最基础的可用性指标。

**生成层指标**：

1. **Faithfulness**：生成的回答是否忠实于检索到的上下文（不编造）
2. **Answer Relevancy**：回答是否切题
3. **Context Precision**：检索结果中有多少是对生成有用的（噪声比）

**评估方法**：

1. **人工标注**：金标准，但成本高。适合建 benchmark 集。
2. **LLM-as-Judge**：用 GPT-4 / Claude 打分。便宜快速，但有偏差需要校准。
3. **A/B 测试**：线上对比不同策略的用户满意度。
4. **RAGAS 框架**：开源 RAG 评估框架，自动计算 faithfulness、relevancy 等。

工程实践：
- 建立 golden test set（50-100 条 query-doc 配对），每次改动跑一遍回归
- 检索和生成分开评估，否则生成好可能掩盖检索差
- 关注 tail case：那些检索完全失败的 query，比平均指标更能暴露问题
